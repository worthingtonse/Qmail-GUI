- QMailDashboard refresh crash: `handleRefresh` calls `checkForNewMail()` even though that function is commented out/undefined in QMailDashboard.jsx (around the refresh handler). Clicking Refresh throws a ReferenceError and stops the UI from updating.
- USB check never passes: USBCheckScreen initializes local `usbStatus` from the `isUSBDrive` prop once and never updates it when the prop changes. When Wallet detects a USB drive later, the screen still shows the failure state because `currentUSBStatus` stays false.
- Send-status polling stuck: ComposeModal.pollTaskStatus reads `result.data.status`, but `getTaskStatus` returns `state` (with progress flags). Tasks with a `task_id` never reach the “completed” branch, so sending stays in a perpetual sending state even after the backend finishes.
