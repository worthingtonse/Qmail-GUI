API alignment (system/wallet endpoints)
- /system/config/usb: GUI calls this and maps `on_usb`竊蛋onUsb`, `root_path`竊蛋rootPath`; Core returns `required:false` hardcoded. Shapes mostly align, but GUI ignores HTTP status/`status` and will silently treat failures as `{onUsb:false}` rather than surfacing errors.
- /system/tasks: GUI expects flat `{task_id,status,progress,message,data}`. Core returns `{status:"success", payload:{id,status,progress,message,data,raida_responded,elapsed_ms,timeout_ms}, raida_total}`. Because GUI never drills into `payload`, successful responses look invalid and get reported as errors.
- /system/dropdown: GUI accepts any filename and expects `{status,operation,file,path,items[]}`. Requests outside the whitelist 400, and even allowed files need field remapping (`lines`竊蛋items`).
